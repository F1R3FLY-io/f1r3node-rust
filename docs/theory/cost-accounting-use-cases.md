# Cost-Accounted Rho Use-Case Coverage

**Status:** Implementation-aligned traceability catalog
**Scope:** Formal proof anchors, finite-state model anchors, and Rust test
targets for the cost-accounted rho migration.

This catalog is the acceptance map for hardening the implementation. Each
use case binds an operational scenario to the formal result that justifies it
and to the Rust test target that exercises the production boundary. The
external paper remains unchanged; this document records repo-local coverage.

## Test Execution

Rust cost-accounting tests are run with `cargo nextest run` so tests that
observe process-local metrics do not collide through shared in-process state.
The core hardening targets are:

```bash
cargo nextest run -p rholang accounting::
cargo nextest run -p rholang generated_frontier_v12
cargo nextest run -p rholang generated_frontier_v13
cargo nextest run -p rholang -E 'test(empty_cost_trace_still_has_consensus_digest) | test(parallel_interpreter_cost_trace_digest_is_repeatable) | test(oversized_runtime_event_is_rejected_without_trace_entry) | test(nonbillable_frames_do_not_enter_cost_trace) | test(cost_trace_digest_domain_separates_event_kind_path_redex_index_and_multiplicity)'
cargo nextest run -p casper runtime_manager_test::
cargo nextest run -p casper -E 'test(replaycomputestate_should_require_cost_trace_after_activation) | test(replaycomputestate_should_catch_cost_trace_event_count_mismatch) | test(oop_replay_cost_trace_survives_failed_deploy_rollback) | test(mixed_success_and_oop_deploys_keep_isolated_cost_traces)'
cargo nextest run -p casper -E 'test(replay_payload_hash) | test(block_hash_changes_when_processed_deploy_cost_trace)'
cargo nextest run -p casper cost_accounting_v12
cargo nextest run -p casper cost_accounting_v13

(cd formal/tlaplus/cost_accounted_rho && tlc MCRuntimeBudgetReplay.tla -config RuntimeBudgetReplay.cfg -workers auto -nowarning)
(cd formal/tlaplus/cost_accounted_rho && tlc MCCostAccountingThreats.tla -config CostAccountingThreats.cfg -workers auto -nowarning)
```

use-case/property tests for the production APIs that implement the design.
