# D3 — Replace Phlo with Tokens (approved design)

**Status:** USER-APPROVED (plan-then-confirm; all 4 open decisions resolved as recommended). **Binding contract for the D3 implementation agent.** Spec (LAW): `publications/cost-accounting/cost-accounted-rho.tex`. Conforms to DR-6/DR-9/DR-11/DR-13. **Governing principle: spec-minimalism** — D3 is REMOVAL + REALIGNMENT (delete the non-spec singular-phlo escrow/price model; make consensus cost = the per-COMM token count); add NOTHING beyond DR-9 + the spec.

## Resolved open decisions (user-approved)
- **OD-1:** accepted deploys are **unmetered-for-liveness** — the gate already proved fundedness; the runtime's existing AST/term-count limit handles non-termination; NO per-deploy budget cap (the spec has none). Remove the `Cost::create(phlo_limit,…)` cap; use the existing `unmetered()`/`set_unmetered`-style path for the user-deploy reduction (it stays metered for DIAGNOSTICS, but no OOP-abort budget gates an accepted deploy).
- **OD-2:** D3 + D4.1 land as **ONE atomic commit** (the only build-green unit; the escrow data model and its precharge/refund consumer are circularly coupled).
- **OD-3:** split `BillableKind` into **`Comm`** (send/receive) vs **`Reduction`** (new/match/if) — internal only, not on the wire; preserves diagnostic fidelity.
- **OD-4:** **delete `phlo_share` outright** (no spec construct maps to it; per-component Split/Join spend is task #12's `effective_supply` closure, not a share field). Do NOT zero-and-reserve.

## D.1 — Remove the singular-phlo escrow
Delete `DeployData.phlo_limit`/`phlo_price`; all escrow arithmetic (`checked_total_phlo_charge[_value]`, `refund_amount_for_token_cost[_value]`, `total_phlo_charge`, `validate_phlo`, casper_message.rs:1116-1197); `Cosigner.phlo_share`, `Cosigned::total_phlo_share`, the `phlo_limit` param of every `Cosigned::from_*`, the share-related `CosignedError` variants; `ProcessedDeploy.primary_phlo_share` + `try_refund_amount`. **Wire (DR-6 fresh-genesis, no back-compat):** remove `DeployDataProto.phloPrice(7)`/`phloLimit(8)`/`primary_phlo_share(15)`, `CompoundSigner.phloShare(4)`, `SigAtom.phloShare(4)`; add `reserved` for the retired tags (precedent: CasperMessage.proto:291). **Nothing replaces them** — the per-signature token supply `Σ⟦s⟧` (balance datum on `from_sig(s)`, DR-13) IS the funding; the gate is the cost authority (Def 19, §7.6). KEEP the cosigner list/sort/dedup/per-signer verification/threshold-algebra (DR-10) — orthogonal to escrow.

## D.2 — Demote `costs.rs` per-op gas to DIAGNOSTIC (DR-9)
Consensus consumed cost = **count of `Comm` events** (each = 1). One surgical change: `reconcile_lane` (accounting/mod.rs:667-680) accumulates `consumed_units += (kind == Comm) ? 1 : 0` (was `+= rec.event.weight`). `Reduction`/`Primitive`/`Substitution` events still walk into `committed`/`cost_amounts` for diagnostics but contribute 0 to consensus. `total_cost()` now returns the per-COMM token count. The rich per-op weights stay in the diagnostic event log + (pre-OD-1) the liveness gate. Per §7.2 + Worked-Example Steps 1-3 (tex 2750-2779): one token per COMM.

## D.3 — Flip the consensus count to per-COMM (CENTERPIECE; the D1→D3 handoff)
Three counters move to per-COMM (sends+receives only) **in lockstep**:
- **(a) Runtime** (OD-3): `reduce.rs` keeps emitting a billable event at `send`/`receive` (→ `BillableKind::Comm`) and at `new`/`match`/`if` (→ `BillableKind::Reduction`); D.2's reconcile counts only `Comm`. So consensus consumed = COMM count (8 for §7.4, not 9).
- **(b) Gate** `delta_sigma::demand()` → per-COMM: stop `.plus_one()` for `New`/`Match`/`If`/conditional nodes (still recurse into their process-position bodies). `known_lower_bound` now == the test's `comm_node_count`. Delete the "NOT comm_node_count … D3 flips both" comment at acceptance.rs:233-236.
- **(c) Settlement debit** → per-COMM (automatic: `acceptance.rs:414-415` uses `demand.known_lower_bound`; the `post Σ⟦s⟧ = pre − ΣΔ` write rides `CloseBlockDeploy::post_eval`).
**Replay/consistency:** `demand` is a pure deterministic AST pass; the gate reads `Σ_s` from the deterministic merged pre-state; `replay_admission_mismatch` recomputes with the same per-COMM `demand`; `replay_cost_mismatch` asserts `processed.cost == total_cost()` (now per-COMM both sides). The `Δ_s == consumed` bridge re-pins per-COMM across gate/runtime/settlement. §7.4 re-pins 9→8; App.B handler = 6.

## D.4 — D3+D4.1 atomic (OD-2) + runtime budget cap (OD-1)
`phlo_limit`/`price`/`share` are USED by `play_deploy_with_cost_accounting_cosigned` (runtime.rs:578-798: precharge→user→refund) + its replay twin (replay_runtime.rs:370-496) + `evaluate_cosigned`'s `Cost::create(phlo_limit,…)` cap (1553). Removing the fields requires removing this fan-out — so **merge into one atomic commit**. Steps: (1) rewrite `play_deploy_with_cost_accounting_cosigned` to gate-funded (no precharge/refund loops; KEEP the inner soft-checkpoint for failed-deploy rollback; user deploy runs, `cost` = per-COMM `total_cost()`); (2) same on replay_runtime.rs; (3) delete `costacc/{pre_charge_deploy,refund_deploy}.rs`, `system_deploy_util` precharge/refund seeds, PoS.rhox `chargeDeploy`/`refundDeploy` (KEEP `sysAuthTokenOps`/`createUnfVault`); (4) remove the `phlo_*` fields (D.1); (5) reconcile `MultiSignerRefinement.v` `pos_charge`/`pos_refund` (keep distinctness lemmas). **Runtime cap (OD-1):** accepted deploys run unmetered-for-liveness (no `phlo_limit` cap); rely on the gate (fundedness proven) + the existing AST/term-count non-termination guard.
