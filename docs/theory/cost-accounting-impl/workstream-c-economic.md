## Stage B — minting (bond + epoch)

> **Governed by [stageb-minting-halt-interface.md](stageb-minting-halt-interface.md)** (the authoritative StageB design; supersedes this sketch where they differ — esp. `mintedEpochs: Set[(Pk,Int)]`, the `CloseBlockDeploy::post_eval` `Σ⟦v⟧`-write seam via a new `SystemDeployTrait::post_eval` hook, the genesis block-1 `initialPhlogiston` path, and the Stage-C halt interface: slash drains `@W_v` + sets `mintingHalted` + zeros `Σ⟦v⟧`).
**LANDED (Stage B).** The dual-write is realized exactly as the authoritative design specifies; this sketch is reconciled to match.

- **Mint at bond:** `bond` is a USER deploy (no `sysAuthToken`), so it does NOT mint inline — it records stake + installs an empty-`@W_v` `VB` (the DR-3 halt until funded). The next authorized `closeBlock` mints `$$initialPhlogiston$$` (block-1 / genesis bonded set) or `$$epochPhlogiston$$` (epoch boundary) to newly-bonded, eligible validators (epoch-keyed ⇒ idempotent). All minting stays in the `sysAuthToken`-bearing path.
- **Dual-write (I-DUAL):** the Rholang half (`mintPhlogiston`→`@W_v` MakeMint purse) and the Rust half (`CloseBlockDeploy::post_eval`→`Σ⟦v⟧`) both derive from the SAME `amount` literal in the SAME close-block deploy. The `closeBlock` fold (`mintPhlogistonToValidators`) publishes the `[(pk, amount)]` mint list onto a Rust-known, user-unforgeable env channel (`sys:casper:mintList`); `post_eval` reads it and mirrors each amount to `Σ⟦v⟧ = from_sig(Ground(pk))` via `supply::produce_balance` (read-modify-replace, single datum). `post_eval` runs IDENTICALLY on play (`RuntimeOps::play_system_deploy`) and replay (`replay_block_system_deploy`) — the consensus-critical symmetry.
- **Epoch mint:** in `closeBlock`'s epoch branch (post-`pickActiveValidators`), the shared `mintPhlogistonToValidators` fold over `allBonds` mints `$$epochPhlogiston$$` to each `active ∧ ¬mintingHalted ∧ ¬mintedEpochs.contains((pk,epochIndex))` validator. The genesis block-1 path runs the SAME fold on the non-epoch branch (lifted to `runMVar`) at `epochIndex=0` with `$$initialPhlogiston$$`. Slush grants = the same `mintPhlogiston` from an authorized path.
- **`mintedEpochs: Set[(Pk,Int)]`** (per-validator-per-epoch — NOT `Set[Int]`, too coarse for bond-catch-up). `epochIndex = blockNumber/$$epochLength$$`; mint only if `(pk, epochIndex) ∉ mintedEpochs` (no-op second time). Content-addressed Produce handles replayed-identical; the epoch-key guard handles amount-correctness. `MintingInjection.v` `epoch_mint_idempotent_on_balance` proves a re-minted epoch key is a balance no-op.
- **`generate_epoch_mint_deploy_random_seed` (Stage A) is now DORMANT** — minting folds into `closeBlock` rather than a standalone deploy; the seed is harmless and retained for a future slush-grant deploy. The `post_eval` produce's `random_state` is derived from the close-block deploy's replay-stable `initial_rand` advanced per validator (sorted-pk order), byte-identical play/replay.
- **Threat/UC rows:** UC-CA-153 (epoch mint funds active validators), UC-CA-154 (bond-then-first-close + genesis block-1); TM-CA-155 (unauthorized mint), TM-CA-156 (halted residual funding), TM-CA-154 (mint-replay, DR-13).

## Stage D — fee `Exchange(c,v)` + collection + `!VB` economic loop (DR-4) — IMPLEMENTED
Authoritative design: `staged-fee-exchange.md`. The validator economic feedback loop landed as three green commits (Exchange + genesis wiring; F_v collection + conversion + PoS.rhox; formal; TLA+/Sage/docs). **Purely additive — the committed D2 gate/settlement is UNCHANGED (cost ≠ fee, D.0).**

- **`Exchange.rhox`** (NEW `casper/src/main/resources/`, registered at `rho:lang:exchange`): the spec's conserving 1:1 swap as a persistent JOIN over count-datum carrier channels — `for (@cDatum <- cCarrier & @vDatum <- vCarrier) { cCarrier!(vDatum) | vCarrier!(cDatum) | ret!((true, Nil)) }`. Wired EXACTLY like `capabilities_registry` (`RegistrySigGen` → `$$exchangePubKey$$`/`$$exchangeSig$$`; `EXCHANGE_PK`/`EXCHANGE_TIMESTAMP`/`EXCHANGE_PUB_KEY` in `standard_deploys.rs`; pushed in `genesis.rs` after `make_mint`; `rho:lang:exchange` shorthand → the derived `rho:id:` URI in `Registry.rho`). DR-4: the join requires BOTH inputs (no one-sided mint). 1:1 peg, no rate param (OD-5). It is the acquisition mechanism #13 clients use to convert `c`-fees to `v`-fuel.
- **`F_v` fee collection + epoch fee→v conversion (the economic loop):** each block, `CloseBlockDeploy::post_eval` credits the proposing validator's fee pool `F_v` by `block.body.deploys.len()` (the spec's flat `FeeExtract`, one token per processed deploy — OD-3, no business param). At each epoch boundary BEFORE the epoch mint, PoS `convertFeesToValidators` publishes the ELIGIBLE validators on `sys:casper:feeConvertList` (`active ∧ ¬mintingHalted ∧ ¬convertedEpochs`) + records `convertedEpochs`; `post_eval` reads it and converts each eligible `F_v` 1:1 into `Σ⟦v⟧` (backed; DR-4: empty `F_v` ⇒ no credit). `!VB` is UNCHANGED (D.3); the converted `Σ⟦v⟧` headroom replenishes the gate pool between mints.
- **REALIZATION NOTE (grounding adaptation vs the `staged-fee-exchange.md` D.2 sketch — load-bearing, recorded for reconstruction):** the D.2 sketch described `F_v` as a PoS-private Rholang carrier `@(*feeTag, pk)` with a per-block `feeCount` handoff into `closeBlock` and an OD-4 `@W_v` draw-wallet purse mirror. The implementation instead realizes **`F_v` as a Rust-nameable, content-addressed, reducer-unwritable pool** (`supply::fee_collection_channel`, domain-separated from `Σ⟦v⟧`), written ONLY by Rust `post_eval` — identical DR-13 discipline to the StageB mint / WD-D2 settlement. RATIONALE: passing the per-block count INTO `closeBlock` requires a pre-eval Rholang channel seed, which is incompatible with the replay event-rig (the seed produce double-counts against the rigged play event log → `ConsumeFailed` on the production `process_block` replay path). The Rust-pool realization sidesteps the seam entirely and is trivially replay-symmetric (collection amount recomputed from `block.body.deploys.len()` = `terms.len()`; convert amount read from the same pre-state `F_v`). CONSEQUENCE: PoS owns ONLY the conversion ELIGIBILITY + `convertedEpochs` idempotency (it cannot read the Rust `F_v` or write `Σ⟦v⟧`); the OD-4 `@W_v` draw-wallet purse mirror is NOT performed (the convert credits the consensus gate pool `Σ⟦v⟧` only — the load-bearing half; `@W_v` execution fuel continues to come from the epoch mint). **SETTLED (DR-14, user-ratified): the `@W_v` mirror is unnecessary; `Σ⟦v⟧`-only is the permanent, spec-complete realization.** A second-opinion review confirmed the `@W_v` purse *amount* is operationally inert under the s₀-collapse (every consumer reads presence, not quantity — VB drops it with VH=nil, slash discards it), so crediting `@W_v` with the fee amount is a consensus no-op; building the Rust→PoS pre-eval seam to perform it would re-introduce the DR-13(b)-rejected Rholang-exposure of a Rust quantity plus replay-rig fragility for zero effect. No `@W_v` fee-mirror and no `rho:casper:feeCount` seam will be built (it would matter only under a reverted literal per-COMM `VB`-draw model, which DR-11 rejects on performance grounds).
