## Stage B — minting (bond + epoch)
- **Mint at bond:** `bond` is a USER deploy (no `sysAuthToken`), so DON'T mint inline. **B.1a (preferred):** `bond` records stake + installs `VB`; the first `closeBlock` after the bond mints `$$initialPhlogiston$$` to newly-bonded, unfunded validators (epoch-keyed ⇒ idempotent). Keeps all minting in the `sysAuthToken`-bearing path.
- **Epoch mint:** in `closeBlock` epoch branch (PoS.rhox:621-749), add a step folding over `allBonds`: for each `active && !mintingHalted`, `mintPhlogiston(pk,$$epochPhlogiston$$,sysAuthToken,…)` (fold like `commitCurrentEpochRewards`:668). Slush grants = the same `mintPhlogiston` from an authorized path.
- **Multi-parent merge idempotency (central risk):** add `"mintedEpochs":Set[Int]` (or per-validator `lastMintedEpoch`) state guard — `epochIndex = blockNumber/$$epochLength$$`; mint only if not already minted (no-op second time, mirror slash idempotency PoS.rhox:563). Content-addressed Produce handles replayed-identical; the epoch-key guard handles amount-correctness. **New `MintingInjection.v` lemma:** a second `AMint` of the same epoch key is a wallet no-op (so `admin_trace_net_increase_bounded_by_minted` bounds fuel by Σ over DISTINCT (validator,epoch), not by #parents).

## Consensus-safety (cross-cut, §6.3/§6.4)
Every new unforgeable name (`@W_v`, quarantine, slot) + every mint/slash/redeem is a consensus quantity: deterministic over replay-stable inputs (content-addressed by pubkey / domain-tagged seeds), recomputed in replay, gated by the unforgeable `sysAuthToken`/PoS-multisig root. Top fork hazards: R-1 name determinism (content-addressing + injectivity lemma + two-order tests; never `primary().sig`); B.3 epoch-mint double-mint under merge (epoch-key guard + `MintingInjection.v` idempotency lemma); slash↔merge recovery (preserve the existing idempotency guard); `$$posPubKey$$` latent substitution.
