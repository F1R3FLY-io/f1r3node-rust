# 13 · Scenario corpus generation

## 1 · Family motivation

The findings generated by all of the previous families need to be
**captured** as deterministic, replayable fixtures so that
regression testing remains stable across runs and machines. This
family defines the canonical scenario schema and operates the
corpus generator that produces JSON fixtures from the search engines.

## 2 · Models in this family

| Model                                                                                                                | Purpose                                                                                                   |
|----------------------------------------------------------------------------------------------------------------------|-----------------------------------------------------------------------------------------------------------|
| [`scenario_schema.sage`](../../../../../formal/sage/slashing/scenario_schema.sage)                                   | Defines the JSON schema every fixture must conform to: action types, observable contract, metadata fields |
| [`scenario_search/corpus_generator.sage`](../../../../../formal/sage/slashing/scenario_search/corpus_generator.sage) | Drives the search engines and writes conforming fixtures to disk                                          |

## 3 · The canonical schema

```json
{
  "$schema": "scenario_schema_v1",
  "fixture_id": "uuid",
  "fixture_kind": "string (e.g. hypothesis_adversarial_scheduler)",
  "seed": "string (hex)",
  "search_engine": "string",
  "search_parameters": {
    "n": "integer",
    "depth": "integer",
    "budget": "integer"
  },
  "actions": [
    {"op": "bond|sign|equivocate|slash|withdraw|rebond|propose|dispatch|...",
     "v": "validator-id", "...": "op-specific fields"}
  ],
  "expected_invariants_at_end": {
    "invariant_key": "bool"
  },
  "classification": {
    "threat_class": "bisimilar|permitted_bug_fix|candidate_boundary|projection_risk|assumption_counterexample|unexpected",
    "ledger_status": "confirmed_current_bug|confirmed_fixed_bug|...|needs_source_audit",
    "cited_bug": "Bug #N (optional)",
    "cited_theorem": "T-XX (optional)"
  }
}
```

Every fixture in
[`casper/tests/slashing/`](../../../../../casper/tests/slashing/)
JSON-driven tests (`*_fixtures`, `*_replay`, `*_corpus`) conforms
to this schema. The schema enforces the **classification rule**
from [`../pipeline/01-witness-to-source-rule.md`](../pipeline/01-witness-to-source-rule.md)
mechanically — a fixture without classification fails the JSON
validation step.

## 4 · The corpus generator

```
algorithm corpus_generator(engines : List(SearchEngine),
                            budgets : Map(SearchEngine → ℕ),
                            outputs : Path) → CorpusStats:
    let stats ← empty()
    for each engine in engines:
        let witnesses ← engine.search(budget = budgets[engine])
        for each w in witnesses:
            let fixture ← format_per_schema(w)
            validate_against_schema(fixture)        (* halt if invalid *)
            classify_or_halt(fixture)                (* must not be unexpected *)
            write(outputs / fixture.fixture_id + ".json", fixture)
            stats.append(engine, fixture)
    return stats
```

The generator is invoked from
[`scripts/ci/slashing-search-horizon.sh`](../../../../../scripts/ci/slashing-search-horizon.sh)
at the `frontier`, `nightly`, and `exhaustive` tiers; output goes
to `target/slashing-search-horizon/` and is replayed by the
Rust-side fixture tests.

## 5 · Promotion targets

This family does not directly promote witnesses to theorems; it is
the *plumbing* that delivers other families' witnesses to the
Rust-side regression layer. The promotion happens at the receiving
end:

- [`casper/tests/slashing/hypothesis_rust_replay_fixtures.rs`](../../../../../casper/tests/slashing/hypothesis_rust_replay_fixtures.rs)
  replays every fixture on the production code path.
- [`casper/tests/slashing/minimal_counterexample_catalog_replay.rs`](../../../../../casper/tests/slashing/minimal_counterexample_catalog_replay.rs)
  replays the *minimized* corpus to keep CI fast.
- [`casper/tests/slashing/horizon_search_fixtures.rs`](../../../../../casper/tests/slashing/horizon_search_fixtures.rs)
  and `horizon_v2_search_fixtures.rs` consume the horizon-tier output.

## 6 · Methodology note

The corpus-generation family is the methodology's **bookkeeping**
layer. Every Sage / Hypothesis finding flows through here on its way
to the Rust regression suite; the schema enforces classification at
the boundary. Without this layer, the methodology's witness rule
would be a paper promise; with it, the rule is mechanically
enforceable.
