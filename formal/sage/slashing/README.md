# Sage support models for slashing

This directory contains optional Sage scripts for finite modeling of Casper slashing behavior. These scripts are not proof authority. They are used to search small exact models, produce witnesses, and validate theorem candidates before the claims are promoted to Rocq or TLA+.

The models use Sage exact integers, sets, combinatorics, graph APIs, schedule permutations, and integer vectors. They do not justify new Rocq axioms, and their output must not be cited as a proof of protocol correctness.

## Scripts

`closure_model.sage` models two-level slashing closure with Sage `DiGraph`, `Subsets`, `Integer`, and `binomial`. It computes the least closure generated by direct equivocators and directed neglect edges, checks the iterative closure against Sage transitive closure, and finds quorum-loss witnesses when the closure exceeds the configured fault bound.

`tracker_race_model.sage` models finite tracker schedules with Sage `DiGraph`, `Permutations`, and `Set`. It produces a minimal lost-update witness for the pre-fix read/write model and checks that atomic read-modify-write schedules keep every observed hash.

`pipeline_effect_model.sage` enumerates small bond maps and slash sets with Sage `cartesian_product`, `Subsets`, `Integer`, `ZZ`, and `vector`. It checks exact accounting, idempotence, zeroing of slashed bonds, preservation of unslashed bonds, non-negative balances, and rejected-slash reissue no-op behavior.

`parent_prestate_authorization_model.sage` checks the receive-side split between ambient snapshot bonds, block parent-pre-state bonds, and PoS execution bonds. It also checks that recovered rejected slashes are reissued only when their invalid-block evidence is still current.

`weighted_closure_model.sage` models the same neglect closure with stake weights and searches for active-stake quorum failures.

`damage_optimizer.sage` searches bounded graphs and stake vectors for maximum slashed stake and validator-count amplification under a bounded direct-equivocator stake budget.

`validator_boundary_model.sage` compares current-validator filtering with an unfiltered evidence-domain projection to find stale/off-era evidence boundary cases.

`evidence_visibility_model.sage` models partial evidence visibility, reports, induced neglect edges, and full-visibility accountability gaps.

`graph_edge_cases_model.sage` checks duplicate edges, self-edges, disconnected cycles, and cycles that reach a direct offender.

`bounded_arithmetic_model.sage` compares exact Sage arithmetic against checked, wrapping, and saturating bounded integer projections.

`differential_bisimilarity_model.sage` searches small models for behavior differences and classifies them as unexpected, candidate boundary divergences, or permitted bug-fix divergences.

`quorum_intersection_model.sage` checks weighted quorum intersection over bounded stake vectors.

`closure_certificate_model.sage` emits fixed-point depth and first-slash-round certificates, including shortest neglect paths.

`slash_order_model.sage` permutes batch slash execution orders and checks final bonds, slashed set, vault balance, duplicate reissue idempotence, and zero-bond no-op behavior.

`epoch_lifecycle_model.sage` models current-epoch/current-validator filtering for stale and fresh evidence.

`evidence_propagation_model.sage` models visibility/report stages over time; reports can shrink accountability closure, so the checked invariant is that active neglect edges are visible and unreported.

`arithmetic_envelope_model.sage` computes fixed-width safe envelopes of the form `initialVault + validators * maxBond <= limit`.

`record_normalization_model.sage` checks record meaning modulo insertion order and duplicate hashes, plus recovered rejected-slash normalization by invalid hash after own-detected coverage is removed.

`adversarial_timing_game.sage` searches bounded adversarial choices of direct equivocation, neglect edges, visibility, reports, and stake distribution against objectives such as honest-slashed stake, quorum drop, accountability gap, delay, and damage ratio.

`evidence_timing_attack_search.sage` searches local-view divergence, report-time closure shrinkage, and delayed evidence slash timing.

`epoch_churn_attack_model.sage` models stale evidence, validator rejoin/rebond identity, and pending-slash carryover policy boundaries.

`theorem_assumption_counterexamples.sage` records minimal witnesses that fail when key Rocq/TLA theorem assumptions are removed.

`weighted_stake_optimization.sage` uses Sage `MixedIntegerLinearProgram` with exact enumeration fallback to maximize weighted slash amplification for bounded graph patterns.

`differential_trace_generator.sage` emits concrete traces for bisimilar, permitted bug-fix, and candidate boundary divergence classes.

`implementation_projection_risk_model.sage` models exact-formal-vs-runtime projection hazards: partial batch failure order, noncanonical key serialization, evidence pruning, bounded arithmetic, and duplicate records.

`scenario_schema.sage` defines the shared JSON scenario/fixture schema, coverage-feature extraction, and threat scoring used by the newer DAG, objective, deep-threat, and Hypothesis scripts.

`dag_behavior_model.sage` models production-shaped DAG block traces with direct equivocation, cited evidence, explicit slash targets, report suppression, retention projection, epoch/churn projection, and multi-level reverse-reachability witnesses.

`objective_frontier_model.sage` ranks deterministic witnesses with exact Sage objective vectors, computes a Pareto frontier, and emits high-priority replay fixtures for retention, arithmetic, weighted damage, report suppression, epoch identity, and reachability assumptions.

`adversarial_campaign_model.sage` composes production-shaped DAG derivation, Rust detector totality/distinct-child regressions, multi-node local views, adaptive stake/quorum objectives, exact-vs-runtime projections, differential-oracle rows, mutation/metamorphic variants, and minimized threat-corpus ranking for defensive bug hunting. Its production DAG campaign uses a Rust-aligned latest-message detectability projection; direct-only and broad citation graphs are emitted only as differential projection-risk comparators. Its detector regression witness records Bug #11 as a permitted bug-fix delta: missing pointers contribute `∅`, and duplicate paths to one offender child count once.

`deep_threat_model.sage` runs deterministic Sage threat modeling over graph-reachability attack shapes, MIP-backed stake-damage optimization, minimum attacker stake, maximum quorum loss, withholding/pruning composition, retention/pruning thresholds, epoch/churn identity boundaries, economic arithmetic envelopes, safe-envelope boundary distance, evidence-denial min-cuts, detector-totality bug-fix witnesses, cross-oracle closure consistency, candidate invariant mining, temporal-window synthesis, mutation-oracle detection, rebond identity lifecycle boundaries, record-lifecycle projection risks, closure-depth extremal witnesses, evidence-addition monotonicity, view-merge confluence, minimal slash-basis cataloging, record-key namespace projections, detector traversal termination, detector contribution confluence, closure fixed-point idempotence, report-retention reactivation, no-seed cycle safety, slash-history prefix exactness, neglect-edge orientation sanity, redundant-path evidence-denial cost, slash-target authorization, report namespace isolation, report-antitone closure, direct-seed report dominance, validator-renaming equivariance, bisimilarity delta guarding, minimal counterexample cataloging, and threat-vector ranking.

`horizon_search_model.sage` composes previously isolated frontier axes into cross-coupled horizon witnesses. It models retention windows against gossip plus inclusion delay, proposer withholding, Rust-shaped detector contribution gates, epoch/rebond identity policy, weighted damage outside the closure-bound precondition, partition view merge, checked arithmetic envelopes, and edge-order/matrix-oracle metamorphism.

`horizon_v2_search_model.sage` extends the horizon frontier with Rust-aligned detector DAG projection, multi-record tracker lifecycle, finality-aware evidence retention, weighted damage plus evidence-denial cost, network/era identity boundaries, and exact-vs-projection classification. It uses Sage `DiGraph`, exact `Integer`/`ZZ` vectors, matrix reachability, combinations, permutations, and a MIP objective with enumeration fallback.

`scenario_search/corpus_generator.sage` emits a deterministic scenario corpus across multi-epoch attacks, partial-synchrony view convergence, proposer-schedule liveness, atomic batch semantics, evidence retention/pruning, equivocation-record canonicalization, economic attack optimization, and differential trace classification. It uses Sage graph, set, permutation, vector, rational, and MIP APIs, with enumeration fallback where the MIP solver is unavailable.

`hypothesis_search/hypothesis_scenario_search.sage` uses Hypothesis inside Sage for optional deep property-based and state-machine search. It shrinks witnesses for evidence lifecycle, epoch churn, proposer liveness, batch failure, differential trace, weighted economic, arithmetic, and record-canonicalization risks, then emits deterministic JSON records for review. Its frontier mode adds less-directed novelty/coverage scoring, feature-combination coverage, bundle-based state machines, multi-epoch trace search, adversarial scheduler search, partition/gossip rule state machines, production-shaped DAG trace generation, detector-totality DAG search, cross-oracle closure consistency, adaptive evidence-denial search, composite multi-factor attack search, candidate invariant mining, temporal-window synthesis, mutation-oracle detection, rebond identity lifecycle search, record-lifecycle state-machine search, closure-depth extremal search, defensive adversarial vulnerability campaigns, liveness-as-safety checks, exact-vs-projection differential checks, arithmetic projection stress, generated trace classification, semantic attack campaigns, attack-objective and objective-guided search, metamorphic and Rust-metamorphic property checks, assumption minimization, assumption weakening, precondition fuzzing, Rust-facing differential corpus emission, Rust replay fixtures, evidence-addition monotonicity, view-merge confluence, minimal slash-basis extraction, record-key namespace projection search, detector traversal termination, detector contribution confluence, closure fixed-point idempotence, report-retention reactivation, no-seed cycle safety, slash-history prefix exactness, neglect-edge orientation sanity, redundant-path evidence-denial cost, slash-target authorization, report namespace isolation, report-antitone closure, direct-seed report dominance, validator-renaming equivariance, and bisimilarity delta guarding. Its horizon mode composes retention, delay, fairness, churn, detector, arithmetic, partition, and metamorphic axes into longer campaigns; its horizon-v2 mode adds Rust-aligned detector DAG state machines, record-lifecycle state machines, finality-aware evidence availability, economic objective search, and generated differential classification. Mutable lifecycle/campaign/partition/record/horizon exploration uses Hypothesis `RuleBasedStateMachine`; replayable differential corpora use generated trace strategies.

## Exhaustive bounds

Directed graph enumeration grows quickly. The closure model refuses searches above `--max-models`, which defaults to `1000000`. Increase that limit explicitly if a larger exhaustive run is intended.

For `n = 4`, exhaustive closure enumeration checks all directed neglect graphs and all initial equivocator sets. For `n = 5`, use `--edge-limit` or raise `--max-models` intentionally.

## Trust boundary

Useful Sage findings should be handled as follows:

1. A counterexample can be copied into the slashing docs or TLA+ model as a concrete finite witness.
2. A candidate invariant can be translated into Rocq or TLA+.
3. A correctness claim is accepted only after the Rocq proof is closed without admissions or the TLA+ model check/spec result is reviewed under its finite assumptions.

Sage output is evidence for follow-up work, not a replacement for formal proof.

## Hypothesis

Sage can import Python property-based testing libraries when they are installed in the Sage Python environment. On the current development environment, `hypothesis` is available to `sage -c` as version `6.152.1`.

Hypothesis is useful for optional exploratory scripts that need shrinking and state-machine search over evidence visibility, reports, pruning, epoch churn, proposer schedules, batch failures, network partition/gossip behavior, dropped preconditions, Rust replay traces, horizon and horizon-v2 campaigns, and differential traces. Rule-based state machines are used for mutable evidence lifecycle, bundle-based evidence reuse, multi-epoch churn, partition/gossip scheduling, semantic attack-campaign exploration, horizon campaign exploration, and horizon-v2 detector/lifecycle/availability exploration. Strategies should generate plain Python integers, tuples, and lists, then convert them to Sage `Integer`, `Set`, `DiGraph`, or `vector` values inside the checked property. Quick, deep, stress, and Rust-replay profiles use deterministic no-database settings unless `--persistent-db` is supplied; corpus and corpus-deep profiles use persistent Hypothesis example databases so long-running searches can accumulate frontier examples across runs. Hypothesis output is still test evidence only; deterministic finite Sage witnesses, Rocq proofs, and TLA+ model/spec checks remain the authority for promoted correctness claims.

The supported promotion path is: Hypothesis shrink output, deterministic Sage witness, classified `FINDINGS.md` entry, then Rocq/TLA+/documentation update if the finding changes a theorem, invariant, assumption, or test use case.
